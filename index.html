wpget

Downloads a set of pages from Wikipedia into a local sqlite database.

Welcome to GitHub Pages.

wpget is a tiny Python script that (recursively) downloads a set of pages from Wikipedia into a local sqlite database. Its purpose is to quickly create a local subset of WP articles for further data mining/processing.

$ wpget --recursive Category:Media_companies_of_Germany
$ wpget --lang=de "Axel Springer AG"

Usage

Here's the full usage help text:

usage: wpget [-h] [--recursive] [--lang LANG] [--version] [--database FILE]
             [--content-dir PATH]
             TARGET

retreives a reasonable set of wikipedia pages. limited to 500 pages per
category.

positional arguments:
  TARGET                name of page or category, eg"Barack Obama"
                        or"Category:Presidents_of_the_United_States"

optional arguments:
  -h, --help            show this help message and exit
  --recursive, -r       if a category url is given, wpget will also process
                        sub-categories recursivly
  --lang LANG, -l LANG  retreive pages from different locales than"en"
  --version             show program's version number and exit
  --database FILE, -d FILE
                        specifies which database wpget should write to
                        (defaults to wpget.db)
  --content-dir PATH, -c PATH
                        specifies the path where wpget should store page
                        contents (one file per page). otherwise content will
                        be stored in sqlite database
